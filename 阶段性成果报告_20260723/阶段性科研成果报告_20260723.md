# 围绕 L2 Rollup 跨层 MEV 防御的阶段性科研成果报告

**研究主题：** 围绕 L2 Rollup 跨层 MEV 防御，研究如何在无需可信第三方的前提下，通过密码学机制限制 Sequencer 的排序操纵  
**报告日期：** 2026 年 7 月 23 日  
**报告性质：** 阶段性研究总结；以已完成的理论、证据与探索性数据链路工作为限。

## 一、摘要

本阶段研究围绕 L2 Rollup 场景中排序权、交易可见性与跨层对象生命周期之间的关系展开。研究起点来自 Hybrid TEE-Rollup 的异常路径与可验证仲裁问题，以及对 Flash Boys 2.0、BEV 量化、Rollup 挑战机制等工作的持续阅读。前期工作表明，MEV 的关键并非单一交易策略，而是信息可见性、排序控制和经济激励共同构成的系统性问题；Rollup 虽将执行迁移至 L2，但 Sequencer 仍可能成为影响纳入、延迟和相对顺序的关键角色。

在完成 20 篇文献分类调研后，研究最初将问题定位为“L2 Rollup 跨层 MEV 防御”。进一步核验文献与对象语义后，发现该表述同时包含对象边界、攻击主体、收益判断和协议设计，若直接进入防御实现，容易把未经验证的可见性或时间关系误写为攻击事实。因此，当前研究将题目收缩为“单 Sequencer L2 Rollup 中跨层交易可见性与排序操纵面：问题定义、威胁模型与最小约束设计”。该收缩并非放弃密码学防御，而是先建立防御应当约束何种可见性、在何种能力条件下才有意义的可证伪基础。

本阶段形成的核心成果包括：构建 L1→L2 message、L2 直接交易和 L2→L1 batch publication 三类对象的边界；提出 RQ1–RQ3 及 Sequencer、外部 searcher、普通用户三角色能力模型；建立“可见性→行动能力→前置状态→有效反事实→净收益”的最小可利用性证据链；制定 V0–V4 分阶段验证路线；提出以 Commit-Reveal 为核心的最小可见性约束草案；并完成一项有限范围的 Arbitrum One 探索性数据链路验证。该预实验只说明特定对象映射和描述性时间关系可被复现，不构成跨层 MEV、排序操纵或获利的证据。当前研究的正式边界仍停留在问题定义、证据门槛和验证设计层面。

**关键词：** L2 Rollup；跨层 MEV；Sequencer；交易排序；可见性；Commit-Reveal；Arbitrum；证据链

## 二、研究背景与问题来源

### 2.1 从 Hybrid TEE-Rollup 到排序安全问题

研究的早期基础是 Hybrid TEE-Rollup 相关工作。此前的重点是高频 DApp 场景下如何在低链上开销下保留异常路径的可验证、可恢复与可仲裁能力。对挑战、超时、数据可用性和恢复机制的分析指出：即使正常路径计算正确，异常路径中的消息延迟、状态暴露和排序关系仍可能影响系统安全。由此，研究视角从“如何证明或恢复一次状态转换”扩展到“谁在何时能看到何种交易信息，以及谁能对执行顺序施加影响”。

第十五至第十七周的组会材料进一步完成了问题背景的铺垫。Flash Boys 2.0 将抢跑、夹击和优先 Gas 竞争说明为排序依赖带来的经济激励；BEV 量化研究表明，排序相关收益可以形成规模化、可观测的链上经济现象；Rollup 机制学习则明确 L1 提供最终安全锚定，而 L2 Sequencer 承担交易收集、排序和执行职责。上述文献和机制学习不能直接推出某一 L2 已经发生跨层操纵，但共同说明：当交易内容可被获取、相对位置可被影响且存在可执行策略时，排序权需要被视为一个安全变量，而非单纯的性能实现细节。

### 2.2 从 MEV 量化到 L2 单 Sequencer 的问题交叉

第十八周的工作完成了 20 篇文献的初步收集与分类，覆盖 MEV 基础量化、跨域 MEV、L2 机制与防御、公平排序协议、近期 DAG/公平排序研究等方向。调研显示，既有工作大致形成四条路径：其一是量化或形式化 MEV 的来源、规模和均衡；其二是通过多方传播、投票或 BFT 共识实现公平排序；其三是通过拍卖、优先通道等经济机制重新分配排序权；其四是通过隐私、加密或应用层约束降低可利用信息。

这一梳理带来的关键认识是，单 Sequencer L2 不能被直接等同于多验证者共识网络。Aequitas、Quick Order Fairness、Wendy 和公平排序 SoK 所依赖的共同传播、委员会或多节点承诺条件，并不是单一排序器天然具备的性质；它们提供了公平性参照，但不能无条件移植。Timeboost 等 auction-based sequencing 则提供了经济排序机制的实证对照：其评价维度可用于理解排序服务的集中度、交易质量和使用行为，却不能替代跨层对象的可见性测量，更不能直接证明跨层 MEV。由此，研究将重点放在“单 Sequencer + 跨层对象生命周期 + 无可信第三方”的交叉区域，并优先考察限制排序前可见性的密码学路径。

## 三、研究方向演进与文献调研基础

### 3.1 方向演进

研究方向经历了三个层次的收敛。第一阶段是知识准备：围绕 Rollup、争议解决、MEV 攻击和理论模型建立共同语言。第二阶段是切入点识别：通过 20 篇文献调研，将“排序权的经济化”与“L2 单 Sequencer 的实现现实”结合，形成跨层 MEV 防御的初始目标。第三阶段是研究问题收缩：在定向核验跨域 MEV SoK、L2 MEV 缓解综述、Timeboost、MEV 理论与公平排序文献后，认识到“跨层 MEV 防御”本身还不是可直接验证的对象，必须先区分 ordering domain、对象方向、角色能力和 realized MEV 的证据条件。

当前题目因此调整为“单 Sequencer L2 Rollup 中跨层交易可见性与排序操纵面：问题定义、威胁模型与最小约束设计”。调整后的问题不预设攻击已经发生，也不预设某种密码学机制必然有效；它首先要求回答：L1 发起并在 L2 创建或执行的对象，在排序承诺前可能对哪些角色可见？若某角色同时拥有协议允许的纳入、延迟或相对排序能力，是否存在需要继续检验的潜在操纵面？

### 3.2 定向文献带来的方法约束

文献不是用来为既定方案背书，而是用来限定可说与不可说的结论。跨域 MEV SoK 提供了 single/cross-domain 与 potential/realized 的分类轴，提示本研究必须先识别排序域和控制状态变更顺序的角色，再讨论跨层关系。Theory of MEV I 强调，重排、插入或删除的收益判断必须比较前置状态、允许动作、原始与反事实序列、有效执行以及扣除成本后的净收益。BEV 量化研究提醒，在缺少 ground truth 时，模式统计不应自动上升为攻击事实。L2 MEV 缓解综述则用于组织公平排序、隐私/加密、合约保护和排序权重构等方案的适用边界。

因此，本报告将“文献支持的已有认识”与“本项目待验证的主张”分开：文献可以支持 Sequencer 排序作为值得建模的架构变量、支持经济排序机制并非天然消除 MEV、支持可见性与收益之间存在多重中介条件；文献不能替代本项目对特定 L1→L2 对象映射、私有队列可见性、真实 first-seen、反事实执行或净收益的核验。

## 四、当前问题定义、系统边界与对象生命周期

### 4.1 研究对象与系统边界

本研究以 Ethereum L1 与 Arbitrum 类 optimistic Rollup 的抽象系统为背景，主对象是由 L1 发起、后续在 L2 创建、调度或执行的 message/transaction object。L2 直接提交交易作为对照对象，用于区分直接 L2 排序和跨层生命周期；L2 batch 向 L1 发布作为辅助对象，用于解释 L2→L1 publication 关系。三者不能相互替代。

特别需要区分的是，L1→L2 message 的问题是用户对象何时成为可创建或可执行的 L2 对象；batch publication lag 描述的则是已经形成的 L2 历史何时被发布到 L1。二者方向、语义和可见性含义不同。即使某个 batch 在 L2 执行后才发布到 L1，也不能反推出某条 L1 message 在进入 L2 前经历了相同的时间窗口，更不能由此推出任何主体已经利用该时间差获利。

### 4.2 生命周期与可观测性

主对象生命周期可概括为：L1 构造/提交 → L1 入块 → 排序前可见性阶段 → L2 创建 → L2 执行 → L2 batch 发布到 L1。历史 RPC 可以核验已上链的 block、receipt、event 与最终状态；但它通常不能恢复公共 mempool 的真实 first-seen，也不能读取 Sequencer 私有队列。因此，“排序前可见性”是需要结合协议实现和提交接口核验的系统假设，而不是可由事后区块时间戳直接观察到的事实。

下表给出对象和阶段的最小区分。

| 阶段/对象 | 可由历史链上数据核验的内容 | 本研究的使用边界 |
|---|---|---|
| L1 入块 | L1 transaction、event、block timestamp、状态 | 可作为公开锚点；不等于 first-seen |
| 排序前阶段 | 无法直接恢复私有队列或真实到达顺序 | 对 Sequencer 可见范围须经 V0 核验 |
| L2 创建/执行 | L2 receipt、event、block、状态转换 | 可核验对象是否已出现及执行结果 |
| L2→L1 batch 发布 | batch event、区间映射、发布时间 | 仅描述 publication relation，不作为用户主窗口 |

## 五、研究问题、威胁模型与最小可利用性见证

### 5.1 RQ1–RQ3

本阶段形成三个递进研究问题。RQ1（可见性）：跨层对象在不同生命周期阶段分别对 Sequencer、外部 searcher 和普通用户可见什么信息？RQ2（可利用性）：已知信息在何种行动能力、前置状态和成本条件下才能转化为正的反事实收益？RQ3（约束设计）：若存在待约束的可见性—排序能力组合，最小机制应隐藏什么、保留什么功能并承担何种代价？三个问题对应从对象定义到收益判断再到防御评估的链条，避免用机制设计替代前提验证。

### 5.2 三角色能力模型

Sequencer 被赋予协议允许范围内的纳入、延迟和相对排序影响能力，但不默认其恶意、自营或与 searcher 共谋；若要主张插入自营交易或协作获利，必须单独建模资金、策略、权限与执行路径。外部 searcher 只被赋予基于公开链上信息提交自身交易的能力，不被默认赋予私有队列可见性、删除他人交易或重排他人交易的权限。普通用户拥有自身 payload、提交结果与公开状态，其行动通常受入口接口约束，不被假设具备搜索或排序能力。

该模型遵循三条纪律：信息不等于控制；控制不等于收益；观察不等于归因。单 Sequencer 的存在只说明存在一个潜在排序控制点，不足以说明 MEV 已经发生。历史上观察到的交易位置、地址关联或 revert 也不足以识别具体主体使用了何种能力。

### 5.3 最小可利用性见证

当前威胁命题是条件性的：**若 Sequencer 在承诺相对顺序前可读取会影响 L2 状态的跨层对象内容，且其在协议允许范围内能够改变该对象的纳入、延迟或相对位置，则需要进一步检查是否存在有效、扣除成本后仍为正的反事实收益。** 该命题不是对恶意行为、共谋或既有利润的事实判断。

为避免把“时间差”或“可见性”直接命名为 MEV，本研究建立最小可利用性见证：第一，明确对象在某阶段对具体角色的可见信息；第二，明确该角色的合法动作和实际能力；第三，获取可重建的前置状态；第四，给出原始顺序与反事实顺序并验证后者可执行；第五，以统一的费用、价格和执行成本口径计算净收益。任一步缺失，都只能保留为潜在风险、候选行为或不可判定对象，而不能报告为已确认的可利用性。

## 六、阶段性成果

### 6.1 文献与证据体系

已形成研究主张与证据对照表、定向文献核验摘记和可迁移结论记录。其价值不在于增加论文数量，而在于为每个关键表述配置来源、允许的结论强度和禁止的外推。例如，Timeboost 只能作为经济排序机制对照，不能作为跨层窗口指标；公平排序研究说明多方共同传播与承诺的重要性，不能被误写成单 Sequencer 已拥有的协调能力；MEV 理论文献提供反事实收益的必要条件，不能被当作 Arbitrum 上实际利润的实证证据。

同时，研究过程中发现一份标为 Aequitas 的 PDF 实际为无关论文，已记录该文件核验问题并未将其作为 Aequitas 证据。这一处理体现了报告的证据原则：对来源、对象语义与结论范围存在疑问时，宁可保留不确定性，也不将错误材料纳入论证链。

### 6.2 概念与系统模型

已完成问题定义、对象边界、生命周期、术语表和反例设计。主要概念贡献是明确 message visibility 与 batch publication lag 不是同一指标：前者要求按角色和阶段说明对象内容、元数据及关联状态的可获得性；后者仅描述 L2 batch 向 L1 发布的时间关系。模型还通过反例说明，有时间差但无可获利动作、Sequencer 可见但反事实无效、或 batch 发布滞后较长，均不能单独推出 MEV。

### 6.3 威胁模型与验证路线

已完成三角色威胁模型、最小可利用性见证模板、RQ—假设—证据需求对照表以及 V0–V4 验证路线。V0 核验对象语义，确认 message、L2 创建/执行与 batch 的事件和接口粒度；V1 在小样本上验证原始数据链路、唯一关联规则和失败分类；V2 才报告描述性映射与时间差；V3 才可在状态重建和成本计价后讨论候选行为与反事实收益；V4 才评估机制性质、实现接口、延迟和成本。后续阶段不能替代前序阶段缺失的证据。

| 阶段 | 目标 | 达到后可报告 | 仍不可报告 |
|---|---|---|---|
| V0 | 对象语义与时间锚点核验 | 研究对象和生命周期定义 | 可见性长度、攻击或收益 |
| V1 | 小样本链路与映射质量 | 数据链路可/不可行 | 大样本分布、MEV 行为 |
| V2 | 描述性实验 | 已定义指标的描述统计 | 攻击频率、主体归因 |
| V3 | 行为与反事实收益 | 完整证据链支持的个案 | 观察即攻击、普遍因果 |
| V4 | 机制评估 | 特定假设下的性质与代价 | 普适安全、消除全部 MEV |

### 6.4 最小 Commit-Reveal 可见性约束草案

为 RQ3 提供可被否定或改进的技术起点，研究提出最小 Commit-Reveal 草案。其目标是在排序决策前隐藏完整 payload，缩小“可读内容 + 可调位置”这一条件组合的适用范围。承诺可写为 `H(domain || chainId || rollupId || sender || nonce || payload || salt)`，通过域隔离、链/rollup 标识、发送者与 nonce 绑定，降低跨域重放或错误解释风险。最小流程为：提交 commitment → 对 commitment 的相对顺序或 slot 作出可验证承诺 → reveal payload 与 salt → 验证 commitment 和域 → 执行或按超时规则处理。

该草案目前不是已实现协议，也没有给出完整安全证明。其待验证性质包括：在高熵 payload 与随机 salt 前提下的有限隐藏性、reveal 与承诺的一致绑定、未 reveal 不无限阻塞的活性、域隔离与抗重放。它明确不自动解决元数据泄露、低熵 payload 猜测、垃圾承诺、Sequencer 拒绝纳入或延迟、审查、强制纳入缺失、reveal 后重排，以及跨层生命周期兼容性问题。只有在 V4 中将机制目标与已确认的威胁能力对应，并完成原型、安全与性能评估后，才可评价其实际代价和防御范围。

### 6.5 探索性 Arbitrum 数据链路验证及边界

在后续研究收缩前，已完成一项限定范围的 Arbitrum One 探索性数据链路验证。该工作应作为预实验性成果记录，而非当前主线的大规模实证。样本锚定于固定 finalized Ethereum 区间，选择 40 个连续 batch 和 25 条 kind=9 retryable message；前者通过 batch 到连续 L2 block 区间的 `range` 规则关联，后者通过官方语义一致的 retryable ID 与 L2 receipt 的 `exact` 规则关联。结果显示，25 条 retryable message 的 exact 映射为 25/25，40 个 batch 的 range 映射为 40/40，所选 message 的核心 provenance 字段完整率为 100%。

预实验固定了三个描述性时间量：`W_ingest = t(L2 retryable creation block) − t(L1 message finalized block)`；`W_effect = t(L2 successful redeem block) − t(L1 message finalized block)`；`P_lag = t(L1 Sequencer batch inclusion) − t(L2 mapped last block)`。在该 selected sample 中，`W_ingest` 为 404–751 秒，p50 为 498 秒，均值为 551.32 秒；`P_lag` 为 7–39 秒，p50 为 16.5 秒，均值为 17.275 秒。25 条样本均成功 auto redeem，且 redeem 与 creation 位于同一 L2 block，因此本样本中的 `W_effect` 与 `W_ingest` 重合。

这些结果只支持如下描述：在固定 finalized 区间、指定 kind=9 对象和明确映射规则下，L1 event 与 L2 创建 receipt 可以复现关联，batch 可以映射为连续 L2 block 区间，且上述时间差可以由 block hash 审计。它们不表示 public first-seen，不包含 Sequencer 私有队列时间，不等于任何参与者的真实反应窗口，也不构成 Sequencer 偏置、抢跑、攻击频率、地址归属或 MEV 利润的证据。特别是 `P_lag` 是 L2→L1 publication lag，不能与 L1→L2 message 生命周期合并解释。由于 25 条样本均为成功 auto redeem，该组样本也不能代表人工 redeem、失败或未 redeem 的完整生命周期。

### 6.6 版本关系与当前正式边界

材料中存在两个相邻版本：探索性成果目录记录了上述预实验；7 月 17 日后的计划调整目录则将当前正式阶段的交付收缩为系统模型、证据门槛、验证路线和机制草案，并明确 V0/V1 尚未作为当前主线正式执行。两者并不互相否定：前者证明某一对象链路在有限样本中值得进一步核验，后者则避免把该链路验证提前升级为跨层 MEV 结论。

因此，本报告的当前正式结论是：已完成问题定义、系统假设、研究问题、角色能力模型、可利用性见证、验证路线和待验证的最小机制方向；尚未完成可推广的对象语义核验、真实私有阶段测量、前置状态重放、反事实收益计算、协议实现或防御效果评估。

## 七、创新点、预期贡献与当前局限

### 7.1 阶段性创新与预期贡献

第一，提出面向跨层对象的概念区分：将 L1→L2 message visibility 与 L2→L1 batch publication lag 分离，避免把不同方向、不同对象和不同观察条件都称为“跨层窗口”。第二，提出单 Sequencer 场景下的条件性角色模型：不将多方公平排序假设直接移植给单一排序器，也不夸大外部 searcher 的控制能力。第三，提出先对象映射、后描述统计、再反事实收益、最后机制评估的证据驱动流程。第四，提出最小 Commit-Reveal 接口方向，明确其要限制的是排序前完整 payload 的可见性，而不是笼统宣称“消除 MEV”。

这些贡献目前主要属于概念、模型和方法层；机制贡献仍是待验证的设计贡献。它是否成立依赖隐藏性、绑定性、活性、抗拒绝服务、强制纳入兼容性、跨层状态机正确性以及性能成本等后续证据。

### 7.2 当前局限

当前最重要的限制是，历史链上数据无法直接恢复 public first-seen 和 Sequencer 私有队列，因此无法仅凭 block timestamp 判断排序前信息集。其次，尚未对主对象建立经过官方文档、合约或 SDK 完整核验且可跨时间稳定复跑的正式映射规则；探索性 retryable 链路不能自动推广到所有跨层消息类型。再次，尚未获得可重建的前置状态、预注册候选规则和统一成本口径，因而不能进行可靠的反事实收益分析。最后，Commit-Reveal 仅是接口级草案，尚未处理隐私泄露、审查、可用性、费用和用户体验的系统权衡。

## 八、下一阶段计划与可证伪条件

下一阶段按照 V0→V1 的顺序推进。首先，以官方文档、Nitro 合约和 SDK 为一手依据，核验主对象的事件语义、唯一标识、L2 创建/执行关系与时间锚点，明确哪些关系只能得到区间映射或不可映射结论。其次，建立可复跑的小样本链路：保存 raw 与 processed 记录，禁止以“时间最近”替代对象关联，记录 archive/RPC 缺失、事件不完整、ID 派生失败、receipt 缺失和生命周期未终结等失败类别。只有在对象关系、字段完整性和映射规则稳定后，才进入 V2 描述性统计。

如果 V0 发现对象语义无法稳定确定，或 V1 无法在不依赖任意近邻匹配的前提下完成映射，研究将收缩为“跨层可见性与角色能力的概念/方法研究”，而不制造跨层收益结论。如果 V3 无法重建前置状态、构造可执行反事实或定义净收益口径，则仅保留候选行为和不可判定记录，不报告利润。如果 V4 发现 Commit-Reveal 的性质、活性或成本不可接受，则相应缩小机制目标或转向组合机制，而非宣称该方案天然安全。

## 九、结论边界与待验证事项清单

**本阶段可以报告：** 已完成问题定义、对象边界、术语体系、RQ1–RQ3、三角色能力模型、最小可利用性见证、V0–V4 验证路线、字段与失败处理设计，以及待验证的 Commit-Reveal 机制草案；已完成一项有限样本的 Arbitrum 对象映射与描述性时间关系预实验。

**仅能条件性报告：** 若 Sequencer 在相对顺序承诺前可见完整内容且能影响纳入、延迟或相对位置，则存在需要评估的潜在排序操纵面；若 V0/V1 证明对象关系可复核，才可进行描述性验证；若 V3 形成可执行反事实和成本口径，才可讨论个案可利用性；若 V4 证明性质并测得代价，才可评价机制。

**本阶段不得报告：** Sequencer 已恶意、自营、共谋或实际操纵；任一历史时间差等于抢跑窗口、攻击频率或利润；外部 searcher 可见私有队列或重排他人交易；batch publication lag 等于 L1→L2 用户窗口；Commit-Reveal 已安全、低成本、已部署、已抗审查或已降低全部 MEV。

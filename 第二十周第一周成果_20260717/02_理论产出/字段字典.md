# 字段字典

本字典对应本周的真实输出 CSV；每行都保留 provenance、对象语义与映射状态。字段完整性统计针对下列核心 message 字段，在本次 25 条 selected retryable sample 中均为 100%。

| 字段组 | 字段 | 含义与来源 | 用途 |
|---|---|---|---|
| provenance | `l1_block_number/hash/timestamp` | finalized Ethereum block | 固定 L1 时间锚点与重现范围。 |
| provenance | `l1_tx_hash/l1_log_index` | L1 Inbox event 所在 tx/log | 定位原始 JSON 与审计对象。 |
| message | `message_number` | `InboxMessageDelivered` indexed message number | 与 Bridge event 连接，并作为 ticket 派生输入。 |
| message | `sender`, `message_kind`, `payload_hash` | `Bridge.MessageDelivered` 与 Inbox payload | 验证 message 语义；本周只纳入 kind=9 submit retryable。 |
| retryable | `retryable_creation_id` | 官方 SDK 同构的 type `0x69` RLP + Keccak 派生 | 连接到 L2 creation receipt 的唯一 ID。 |
| L2 creation | `l2_creation_block/hash/tx_index`, `creation_receipt_status`, `gas_used` | L2 receipt 与 block | 计算 `W_ingest`、区分 creation success/failure。 |
| redeem | `auto_redeem_tx_hash`, `auto_redeem_block`, `auto_redeem_receipt_status`, `redeem_status` | creation receipt 的 retryable event 与 auto-redeem receipt | 成功时计算 `W_effect`；未发现时不补值。 |
| batch | `batch_sequence`, `before_acc/after_acc/delayed_acc`, `after_delayed_messages_read` | `SequencerBatchDelivered` | batch 连续性与 delayed-message 消费关联。 |
| batch range | `l2_first_block/hash/timestamp`, `l2_last_block/hash/timestamp` | NodeInterface `findBatchContainingBlock` | batch→L2 区间映射与 `P_lag`。 |
| metrics | `w_ingest_seconds`, `w_effect_seconds`, `p_lag_seconds_last_block` | 由上述固定锚点计算 | 仅描述时间关系；不作 MEV 利用判定。 |
| quality | `mapping_status/method/confidence/failure_reason` | pipeline 审计字段 | 强制区分 exact、range、unmapped。 |

## 字段缺失与状态规则

- `exact`：L1 message 到 L2 creation receipt 的 ID 映射成立。
- `range`：L1 batch 到连续 L2 block 区间成立；它不是 transaction-level exact。
- `unmapped`：不计算窗口，必须给出失败原因。
- `W_effect` 只有 successful redeem receipt 时有值；任何无事件、失败或未观测状态都保留为空而非填零。

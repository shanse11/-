## 2. 文献带来的新增认识

- 论文 02 表明：候选行为识别必须同时报告所需字段、启发式和误判风险；无 ground truth 时不能把模式统计直接当攻击结论。

## 3. 本周最终采用的定义

- `W_ingest = t(L2 retryable creation block) − t(L1 message finalized block)`。
- `W_effect = t(L2 successful redeem block) − t(L1 message finalized block)`。
- `P_lag = t(L1 Sequencer batch inclusion) − t(L2 mapped last block)`。

`P_lag` 是 L2→L1 publication lag，不是 L1→L2 主窗口；历史 RPC 不能恢复 L1 mempool first-seen。

## 4. 真实预实验结果

数据区间为 Ethereum finalized blocks `25592648–25594147`，finalized anchor hash 为 `0x89da…5e5e4d`。

| 项目 | 实际结果 |
|---|---|
| 原始 batch event | 123 条；选定连续 batch 40 条，`range` 映射 40/40。 |
| 原始 Inbox event | 53 条；筛出并选定 kind=9 retryable 25 条，`exact` 映射 25/25。 |
| 核心 provenance 字段完整率 | 25/25 的每个核心字段均完整。 |
| `W_ingest` | n=25；min 404 s，p50 498 s，mean 551.32 s，p95 746.4 s，max 751 s。 |
| `W_effect` | n=25；与 `W_ingest` 相同，因为 25 条均成功 auto redeem 且与 creation 位于同一 L2 block。 |
| `P_lag`（last-block anchor） | n=40；min 7 s，p50 16.5 s，mean 17.275 s，p95 28.15 s，max 39 s。 |
| 异常 | selected sample 中无负值、无 `unmapped`、无 receipt 缺失。 |

## 5. 本周结果能支持什么

可以支持：在该固定 finalized 区间内，kind=9 retryable 的 L1 message 与 L2 creation receipt 可经语义明确的 ID 规则精确连接；batch 可映射为连续 L2 block 区间；上述时间差可复算并由 block hash 审计。

不能支持：存在可利用跨层 MEV、Sequencer 偏置、searcher 已在窗口中行动、地址实体归属、攻击频率、MEV 利润，或 Blind Commit-Reveal 的防御效果。

## 6. 是否初步具备进入第 2 周扩样条件

**结论：具备“扩样并固化指标”的初步条件，但尚不具备正式 MEV 行为/利润实验条件。**

理由：本周的对象映射、时间锚点、字段完整性、固定 hash 与重跑链路均已通过 selected sample 的验证；但没有 mempool first-seen、Sequencer queue、同类对照、candidate 行为标签或反事实收益证据。第 2 周应先扩大相同口径的样本并审核映射稳定性，再决定是否加入行为层分析。

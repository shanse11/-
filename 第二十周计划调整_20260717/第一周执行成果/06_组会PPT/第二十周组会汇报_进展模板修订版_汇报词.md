# 第二十周组会汇报词（进展模板修订版）

## 第 1 页｜封面

本周汇报的重点是把原先较宽泛的跨层 MEV 方向，进一步收缩为可见性、排序能力和可利用性三层问题，并补充一篇跨域 MEV SoK 的两页阅读。

## 第 2 页｜主要任务

中长期目标仍是理解单 Sequencer L2 中跨层对象可能受到怎样的排序影响。本周没有做数据统计，而是完成了对象边界、研究问题、验证路线和机制草案的定义工作，使后续实验先有可证伪的输入和门槛。

## 第 3 页｜科研工作进展

主对象限定为 L1 发起、在 L2 创建或执行的 message 或 transaction object。RQ1 问可见性，RQ2 问从可见到收益还缺哪些条件，RQ3 才问最小机制应约束什么。窗口、位置和 batch 时间差都不是 MEV 已发生的证据。

## 第 4 页｜论文阅读（一）

本周引入 Mancino 与 Sevim 的跨域 MEV SoK。它用“single/cross-domain”和“potential/realized”两条轴组织问题，并将 domain 定义为有角色控制状态变更顺序的系统。对我最重要的启发是：先定义 ordering domain，再讨论跨层 MEV。

## 第 5 页｜论文阅读（二）

论文进一步区分潜在价值与已实现收益。后者还会受到竞争、失败、延迟和成本影响。因此本项目不以窗口或相邻交易来宣称攻击，而以可见性、行动能力、状态、反事实顺序和净收益组成见证链。

## 第 6 页｜本周学习体会

最大的纠偏是：L2 batch 向 L1 的发布滞后解释的是 publication 关系，不自动等于用户 L1 到 L2 的主要窗口。Commit-Reveal 目前只是对“排序前 payload 可见性”的待验证约束方向，不代表已经完成防御。

## 第 7 页｜下周科研计划

下周先做 V0 语义核验和 V1 小样本链路预实验设计。只有对象关联和时间锚点稳定，才进入更大样本的描述性研究；否则收缩为可见性与角色模型，不报告攻击频率、收益或协议性能。

## 第 8 页｜结束

我的汇报完毕，敬请老师批评：研究对象的收缩、可利用性见证链，以及 V0/V1 的准入条件是否足以支持下一步的数据验证。

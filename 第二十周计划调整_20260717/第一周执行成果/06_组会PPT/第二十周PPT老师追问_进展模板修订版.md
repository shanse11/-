# 第二十周 PPT 老师可能追问（进展模板修订版）

1. **为什么不直接量窗口长度？**
   因为尚未确认对象是否能稳定映射、锚点是否对应同一生命周期；先量时间差会把不确定的对象关系误写成结论。

2. **单 Sequencer 是否已经说明存在 MEV？**
   不说明。它只是一个潜在排序控制点；还需验证排序前可见性、合法行动能力、可执行反事实顺序和正净收益。

3. **SoK 对你最有用的不是哪一种攻击，而是什么？**
   是把问题放入 `single/cross-domain × potential/realized` 的框架，并把“理论可能”与“经验已实现”严格分开。

4. **为什么 batch lag 不能直接作为跨层窗口？**
   batch 是 L2 向 L1 的发布结果；用户 message 从 L1 到 L2 的创建/执行生命周期可能有不同对象和不同锚点。

5. **Commit-Reveal 能解决审查吗？**
   不能自动解决。草案只尝试约束排序前 payload 可见性，还需处理元数据泄露、未 reveal、审查、强制纳入和 reveal 后重排。

6. **V0/V1 不通过后，研究还剩什么？**
   收缩为可见性与角色能力模型，明确哪些私有阶段无法从历史链上数据验证，而不伪造跨层收益结论。

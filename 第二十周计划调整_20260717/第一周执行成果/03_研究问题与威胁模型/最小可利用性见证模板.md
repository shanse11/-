# 最小可利用性见证模板

> 用途：后续 V3 对每一个候选案例逐项填写。任何空项、无法复核项或反例成立时，都不得将案例提升为“确认 MEV”。

| 要素 | 必填内容 | 最低证据 | 当前第 1 周状态 |
|---|---|---|---|
| 1. 跨层对象 | 对象类型、唯一标识、生命周期阶段 | 官方语义 + 原始链上记录 | 仅定义对象类型，未采集案例 |
| 2. 首次可见阶段 | 角色何时、经何路径获得哪些字段 | 接口/网络或可信日志 | Sequencer 私有可见性待核验 |
| 3. 合法动作 | 该角色可纳入、延迟、排序、提交或不作为的具体权限 | 协议规则/实现 | 仅有角色级假设 |
| 4. 前置链上状态 | 执行前状态根、池子/余额/价格/约束 | 可复现状态快照 | 未获取 |
| 5. 原始顺序 | 已执行交易顺序和 receipt 结果 | 链上 block/receipt | 未获取 |
| 6. 候选反事实顺序 | 调整、插入或删除哪些动作；权限来源 | 明确序列与角色能力 | 未构造 |
| 7. 有效执行 | 原始与反事实是否均满足合约/余额/滑点约束 | 重放或形式化检查 | 未验证 |
| 8. 收益与成本 | 资产变化、统一计价、gas/费用/失败成本 | 可审计计算 | 未定义具体案例 |
| 9. Sequencer 权限/共谋 | 是否必需；若必需，具体假设为何 | 显式模型 | 未假设共谋 |
| 10. 当前证据等级 | 信息/能力/状态/反事实/净收益的完成度 | 本模板逐项核验 | 仅完成定义层 |
| 11. 缺失证据 | 无法得到的日志、状态、接口或计价 | 失败记录 | V0/V1 后确定 |
| 12. 可证伪条件 | 什么事实会使候选失效 | 反例与测试 | 已列：无权限、无效序列、非正净收益 |

固定证据链：

```text
信息可见性 → 行动能力 → 前置状态 → 有效反事实顺序 → 成本扣除后的净收益
```

只有链条完整时，才可以讨论“可利用性”；链条第 1 项成立仅说明可见性风险候选，不能说明攻击或利润。

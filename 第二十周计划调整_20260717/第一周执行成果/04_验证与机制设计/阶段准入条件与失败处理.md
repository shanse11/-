# 阶段准入条件与失败处理

## 准入检查表

| 迁移 | 全部必须满足 | 未满足时的动作 |
|---|---|---|
| V0 → V1 | 对象类型有官方语义；主对象有唯一 ID/官方关联或明确不可映射结论；时间锚点语义一致；batch 与 message 已拆分 | 修正对象范围、补一手资料；仍不成立则只保留概念模型 |
| V1 → V2 | 原始响应可保存；映射不依赖时间最近；失败原因可分类；核心字段完整；同一规则可重跑 | 换对象、改字段、修 RPC 或降低研究粒度 |
| V2 → V3 | 多时间段映射稳定；角色行动能力已定义；前置状态/receipt 能获得；候选规则可预注册 | 停留在描述性研究，不报告攻击频率/收益 |
| V3 → V4 | 明确有待约束的风险条件；能构造反事实和成本口径；机制目标能映射至威胁能力 | 收缩到概念/方法贡献，不实现机制 |

## 失败分类与处理

| 失败类别 | 识别信号 | 处理 | 对研究问题的影响 |
|---|---|---|---|
| RPC 缺少历史数据 | block/receipt/archive 查询失败 | 更换合规 endpoint、缩短区间、保存诊断 | 不能用缺失数据伪造窗口 |
| 事件不完整 | 缺 event/log index/必要字段 | 改用官方事件、保留 unmapped | 缩小到信息完整对象 |
| 对象类型不适用 | 生命周期不产生目标 L2 对象 | 从主样本剔除并说明 | 不外推到该类型 |
| ID 派生失败 | 无官方规则或输入不全 | 只接受官方关联；禁止时间近邻 | 不能称 exact |
| receipt 缺失 | L2 创建/执行记录不可获得 | 标记未确认/未终结 | 不能计算执行或收益 |
| 生命周期未终结 | 没有成功/失败终态 | 保留右删失状态 | 不能将其视为失败或成功 |
| 时间锚点语义不一致 | 混用创建、执行、发布或不同最终性 | 分开字段与指标，重写定义 | 不可比较/不可聚合 |
| 无法获得前置状态 | 无 archive、状态不可重建 | 退回描述性层 | 不做反事实收益 |
| 无法定义反事实净收益 | 无价格/费用/执行路径 | 仅报候选，不报利润 | 不确认可利用性 |

# V1 字段与映射设计表

## 对象选择原则

优先选择可由唯一 ID 派生或由官方事件/接口明确关联的 L1→L2 对象。禁止使用“时间最近”作为配对规则。batch 仅作为 L2→L1 publication 的辅助对象，不用于代替 L1→L2 message 映射。

| 字段 | 来源层/记录 | 用途 | 必填性 | 失效处理 |
|---|---|---|---|---|
| L1 block number/hash/timestamp | L1 block | 固定最终性锚点与可复现区间 | 必填 | 记录 RPC/anchor 失败，不推算 |
| L1 tx hash、receipt、log index | L1 transaction/receipt/log | 定位对象与事件顺序 | 必填 | 标为 event/receipt 缺失 |
| message number / kind | 官方 message 事件/解码 | 候选唯一关联键与对象类型 | 主对象必填 | 类型不适用或解码失败 |
| sender、目标/关键元数据 | 事件/transaction | 后续对象语义和边界检查 | 条件必填 | 记录字段缺失，不补猜 |
| L2 creation receipt / event | L2 receipt/log | 确认 L2 创建对象 | 主对象必填 | receipt 缺失或未找到 |
| L2 execute/redeem receipt | L2 receipt/log | 区分创建、执行、失败、未终结 | 条件必填 | 生命周期未终结/执行记录缺失 |
| L2 block number/hash/timestamp | L2 block | 可复核的链上阶段锚点 | 对 L2 对象必填 | block 不可取或锚点不一致 |
| mapping status | processed | `exact` / `range` / `unmapped` | 必填 | 不适用 |
| mapping method | processed | ID 派生、官方事件关联、官方区间接口 | 必填 | 不适用 |
| failure reason | processed | 失败审计与样本边界 | 必填（失败时） | 不能留空 |

### 映射状态

| 状态 | 含义 | 允许做什么 | 禁止做什么 |
|---|---|---|---|
| `exact` | 唯一 ID/官方事件/receipt 一致 | 对该对象计算语义明确的链上时间关系 | 推断 private first-seen |
| `range` | batch 等对象只定位到区间 | 描述 L2→L1 publication 关系 | 升级为单笔 L1→L2 关联 |
| `unmapped` | 缺唯一证据或对象不适用 | 统计失败类别 | 用临近时间强行补配 |

### V1 建议规模与 golden set

- 20–30 条 eligible message：仅用于验证精确对象关联的可行性。
- 30–50 个 batch：仅用于验证 batch 区间与发布关系。
- golden set：人工核对至少三条连续、语义一致的对象；记录核对字段、核对规则和反例。

以上均为**下一周建议设计**，不是本周数据或承诺的正式实验规模。

# 第二十周汇报词（文档版）

本周我按上周计划，重点完成问题定义、系统假设、研究问题、威胁模型和后续验证路线，而没有提前进入链上实验或协议编码。核心变化是把宽泛的“跨层 MEV 防御”收缩为“单 Sequencer 场景下跨层对象的可见性与排序操纵面”。

我首先区分了 L1 发起、在 L2 创建或执行的 message 与 L2 batch 向 L1 的发布关系。前者才是本研究的主对象；后者只是 L2→L1 publication 的辅助关系，不能被表述为用户消息的主窗口。

接着，我把问题拆为三个层次：对象在何时对谁可见；可见信息需要什么能力、状态和反事实才能形成收益；如果确有待约束风险，最小机制应隐藏什么、保留什么并承担什么代价。这样“窗口存在”不会被直接写成“已有可利用 MEV”。

威胁模型中，Sequencer、外部 searcher 与普通用户拥有不同能力。Sequencer 的排序影响力不能赋给 searcher；而 Sequencer 的排序影响力本身也不证明恶意或获利。

最后，本周设计了 V0 到 V4 的验证路线。下周先核验官方对象语义和小样本数据链路；只有对象映射和字段稳定后，才讨论描述性指标。Commit-Reveal 目前只是一个限制排序前 payload 可见性的机制草案，不声称安全、性能或防御效果。

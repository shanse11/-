# 第二十周结论边界清单

## 可以报告

- 已完成问题定义、系统假设、RQ、角色能力矩阵、验证路线和机制草案。
- 已明确 L1→L2 message visibility 与 L2→L1 batch publication lag 不应混为一个指标。
- 已建立“信息可见性 → 行动能力 → 前置状态 → 有效反事实顺序 → 净收益”的可利用性证据链。
- 已识别 commit-reveal 的目标、元数据泄漏、超时、审查、强制纳入和生命周期兼容性风险。

## 只能使用条件语气

- 若 Sequencer 在排序承诺前可见完整内容且可调节相对顺序，可能需要评估潜在操纵面。
- 若 V0/V1 证明对象关系可复核，才可进入链上描述性验证。
- 若 V3 形成有效反事实与成本口径，才可讨论个案可利用性。
- 若 V4 证明特定性质并测得代价，才可评价机制。

## 本周禁止报告

- 任何链上样本量、窗口长度、映射率、攻击频率、地址集中度或利润；
- Sequencer 已恶意、已自营、已共谋或已操纵；
- 外部 searcher 能看见私有队列或重排他人；
- batch publication lag 就是 L1→L2 用户窗口；
- commit-reveal 已安全、低 Gas、已部署、已降低 MEV 或解决审查。

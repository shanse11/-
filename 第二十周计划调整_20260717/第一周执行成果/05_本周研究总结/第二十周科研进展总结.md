# 第二十周科研进展总结

## 1. 上周计划与本周完成

上周计划要求：（1）继续完善题目对应的问题定义与系统假设；（2）细化研究问题、威胁模型和预期贡献；（3）考虑后续实现与验证路线。本周已完成的实际产出与其一一对应：

1. 题目收缩为“单 Sequencer L2 Rollup 中跨层交易可见性与排序操纵面”，并形成对象、阶段、术语和反例齐全的系统模型。
2. 形成 RQ1–RQ3、Sequencer/searcher/普通用户三角色模型、可利用性见证模板和贡献成立条件。
3. 完成 V0–V4 验证路线、V1 字段映射设计、阶段准入/失败处理和最小 Commit-Reveal 草案。

## 2. 为什么调整题目

“跨层 MEV 防御”把对象、攻击、收益和协议一起打包，容易在缺少对象语义与证据时先承诺防御方案。新题目把问题前移到“谁在何时能看到什么、谁能做什么、何时才值得讨论收益”，使后续研究可逐阶段证伪。题目收缩不意味着否定密码学方向，而是给它增加必要前提：只有被确认的可见性风险才值得被机制约束。

## 3. 研究对象与边界

主对象是 L1 发起、在 L2 创建或执行的 message/transaction object；L2 直接提交交易仅作对照；L2 batch 向 L1 发布只用于解释 L2→L1 publication 关系。batch publication lag 不等于 L1→L2 用户消息主窗口。历史 RPC 能核验已上链 block、receipt、event 与状态，不能直接恢复 mempool first-seen 或 Sequencer 私有队列。

## 4. 三个研究问题与威胁命题

- **RQ1 可见性**：跨层对象在不同阶段对不同角色可见什么？
- **RQ2 可利用性**：可见信息在什么行动能力、状态和成本条件下才可能形成收益？
- **RQ3 约束设计**：最小可见性约束应隐藏什么、保留什么、承担什么代价？

最终威胁命题为：若 Sequencer 在承诺相对顺序前可读取影响 L2 状态的对象内容，且能改变其纳入、延迟或相对位置，则需要检查是否存在有效、扣除成本后为正的反事实收益。命题不假定 Sequencer 恶意、自营或共谋，也不把它当作已发生攻击。

## 5. 新增理论认识与预期贡献

文献核验显示：跨域 MEV 应区分 potential/realized 与不同 ordering domain；公平排序常依赖多方共同传播与承诺；排序约束的效用依赖应用与支付函数；从信息可见到可利用收益需要完整证据链。因此本项目的预期贡献是概念区分、角色模型、分阶段方法和待验证的机制方向，而不是已经完成协议或实证结果。

## 6. 验证路线与最小机制方向

V0 先核验对象语义；V1 仅验证小样本数据链路；V2 才做描述性事实；V3 才可讨论候选行为和反事实收益；V4 才评估机制。Commit-Reveal 草案仅尝试在排序前隐藏完整 payload，并通过 domain separation、salt、reveal 与超时处理使其可验证；它仍会泄漏部分元数据，不能自动解决审查、强制纳入、reveal 后重排或全部 MEV。

## 7. 本周没有开展实验的原因

本周没有正式实验、链上抓取、窗口统计、协议编码、Gas 测试或防御效果测量。这是计划范围要求：没有稳定的对象语义、映射规则与证据门槛之前，实验数字会把不确定的对象关系伪装成结论。本周的实验性产出是验证设计、字段方案、准入条件和失败处理，而不是数据结果。

## 8. 当前局限与下周计划

当前尚不知道具体跨层对象是否能够以官方规则稳定映射，也没有 Sequencer 私有队列、first-seen、前置状态重放或收益证据。下周优先进入 V0/V1：核验官方对象语义、选择唯一关联对象、设置 raw/processed 记录与失败分类；若对象无法精确映射，则收缩为“可见性与角色能力模型”的概念/方法研究，而不伪造跨层收益结论。

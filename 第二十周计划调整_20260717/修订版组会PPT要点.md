# 修订版组会 PPT 要点（6 页以内）

## 1. 封面

题目：**单 Sequencer L2 Rollup 的跨层可见性与排序操纵面**

副标题：问题定义、威胁模型与验证路线设计。

## 2. 本周工作对齐上周计划

- P0：完善问题定义与系统假设；
- P0：细化 RQ、威胁模型和预期贡献；
- P1：只制定实现与验证路线，不报告实验结果；
- 关键纠偏：不把 batch publication lag 当作 L1→L2 主窗口。

## 3. 本项目究竟研究什么

- 研究对象：L1 发起、在 L2 创建/执行的跨层对象；
- 核心问题：排序前可见性在何种能力条件下可能变为排序操纵面；
- 不研究：历史 mempool first-seen、任意链上的统一生命周期、已发生攻击；
- 结论口径：窗口存在不等于存在可利用 MEV。

## 4. 三角色威胁模型

- Sequencer：可影响纳入/相对排序；不能默认恶意或共谋；
- 外部 searcher：仅依赖公共信息和自身提交；不能重排他人；
- 普通用户：可能受影响但不具备策略性排序能力；
- 可利用性还需要可观察信息、允许动作、前态、反事实有效执行和净收益。

## 5. 预期贡献与最小机制方向

- 概念：区分 message visibility 与 batch publication lag；
- 模型：给出可见性—能力—收益的证据链；
- 方法：先做语义和映射，再做行为与收益；
- 机制（待验证）：commit-reveal 仅在排序前隐藏必要内容，揭示后验证承诺；不宣称安全证明或效果。

## 6. 后续验证路线与下周计划

- V0：对象语义核对；V1：小样本链路验证；V2：描述性指标；V3：行为/收益；V4：机制评估；
- 每阶段设置准入条件，未达标则修链路或收缩问题；
- 下周优先 V0 与 V1 的设计准备，实验只在定义和映射规则稳定后启动。
